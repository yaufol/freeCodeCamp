String minus integer

When you try to subtract an integer from a integer-looking string JavaScript casts a new type on the string automatically and do the math:

'5' - 3 //2

As a result the correct result is returned

String plus integer

Imagine that now you need to calculate a sum of a string and an integer:

'5' + 3 //'53'

Voila! Now you have two parts glued to each other. It's called concatenation. Just get used to it. Hopefully it works the same if you swap the places.

Subtracting string from string

Now what do you think will happen if you try to subtract one string from another?

'5' - '3'//2

As JavaScript sees minus sign it supposes that we deal with numbers and tries to convert strings to integers.

Concatenating fake integers vs strings

Accoring to console it is completely okay to put as many + signs in a row. Just take a look:

'5' + + '5'//'55'
'foo' + + 'bar'//'fooNaN'

If you feed the console two integer-looking strings it will concatenate them but if you try to concatenate two strings the same way it will output NaN which is a result of a failed type conversion. What?
